# PedeFácil — Protótipo Navegável no Figma

## Sobre o projeto

O **PedeFácil** é um protótipo de aplicativo mobile para pedidos e entregas de refeições.

O projeto foi desenvolvido como parte da Formação UX Designer da DIO, com o objetivo de aplicar conceitos de prototipagem, design de interfaces, organização de conteúdo e navegação intuitiva.

O protótipo apresenta um fluxo simples de pedido e permite visualizar os restaurantes nos formatos **List View** e **Grid View**.

---

## Objetivo

O objetivo deste projeto é criar uma interface navegável, organizada e fácil de utilizar.

Por meio do protótipo, o usuário pode:

- acessar o aplicativo;
- visualizar restaurantes;
- alternar entre lista e grade;
- consultar o cardápio;
- visualizar detalhes de um produto;
- adicionar um item ao carrinho;
- finalizar um pedido;
- acompanhar a entrega.

---

## Produto escolhido

O **PedeFácil** é um aplicativo fictício de delivery criado para facilitar a realização de pedidos.

A proposta é apresentar as informações mais importantes de maneira clara, como:

- nome do restaurante;
- categoria;
- avaliação;
- tempo estimado de entrega;
- taxa de entrega;
- produtos disponíveis;
- preços;
- valor total do pedido.

Todos os dados apresentados no projeto são fictícios e foram utilizados apenas para fins acadêmicos.

---

## Público-alvo

O aplicativo é direcionado a pessoas que procuram rapidez e praticidade para pedir refeições, principalmente:

- estudantes;
- trabalhadores;
- famílias;
- pessoas com rotina corrida;
- usuários frequentes de serviços de delivery.

---

## Tipo de protótipo

Foi criado um protótipo mobile de alta fidelidade utilizando o Figma.

O projeto apresenta:

- identidade visual definida;
- cores e tipografia padronizadas;
- cards de restaurantes;
- cards de produtos;
- campo de pesquisa;
- botões interativos;
- barra de navegação inferior;
- List View;
- Grid View;
- fluxo navegável entre as telas.

---

## Telas desenvolvidas

### Login

Apresenta os campos de e-mail e senha, além do botão para entrar no aplicativo.

### Página inicial

Apresenta categorias de alimentos, restaurantes recomendados, ofertas e acesso à lista completa de estabelecimentos.

### Restaurantes

Permite pesquisar restaurantes e alternar entre dois formatos de visualização.

#### List View

Os restaurantes são apresentados em uma lista vertical, com imagem e informações detalhadas.

#### Grid View

Os restaurantes são apresentados em uma grade de duas colunas, facilitando a comparação visual entre os estabelecimentos.

### Cardápio

Apresenta informações do restaurante e os produtos disponíveis para pedido.

### Detalhes do produto

Exibe imagem, descrição, ingredientes, adicionais, quantidade e preço do produto.

### Carrinho

Apresenta o produto selecionado, subtotal, taxa de entrega e valor total.

### Confirmação

Informa que o pedido foi realizado e apresenta seus dados principais.

### Acompanhamento

Mostra as etapas do pedido, desde a confirmação até a entrega.

### Perfil

Apresenta informações fictícias do usuário, endereços, pedidos anteriores e configurações.

---

## List View e Grid View

A utilização de List View e Grid View permite que o usuário escolha a forma mais confortável de visualizar os restaurantes.

A **List View** apresenta mais informações em cada item e facilita uma análise detalhada.

A **Grid View** permite visualizar mais restaurantes ao mesmo tempo e favorece uma comparação rápida.

Os botões de troca de visualização ficam próximos ao título da página para facilitar o acesso.
