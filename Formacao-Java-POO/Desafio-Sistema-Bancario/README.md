
# Sistema Bancário com Programação Orientada a Objetos

Este repositório documenta os principais conceitos estudados no desafio de Programação Orientada a Objetos da DIO.

O objetivo do projeto é representar um sistema bancário básico, utilizando os recursos da linguagem Java para organizar entidades, operações financeiras e regras de negócio de forma estruturada.

## Sobre o desafio

A aplicação proposta simula algumas operações comuns de um sistema bancário, como:

- Criação de contas;
- Depósitos e saques;
- Transferências via PIX;
- Criação de investimentos;
- Consulta do histórico de transações;
- Gerenciamento das informações dos clientes.

A atividade foi desenvolvida com foco na aplicação prática dos fundamentos da Programação Orientada a Objetos.

## Conceitos abordados

### Abstração

A abstração permite representar apenas as características e comportamentos necessários de cada elemento do sistema, como contas, clientes, investimentos e transações.

### Encapsulamento

O encapsulamento protege os dados internos das classes e controla o acesso às informações por meio de métodos específicos.

Esse conceito ajuda a evitar alterações indevidas e mantém as regras de negócio centralizadas.

### Herança

A herança permite criar classes que compartilham características e comportamentos comuns.

Em um sistema bancário, diferentes tipos de conta podem herdar atributos e métodos de uma classe principal.

### Polimorfismo

O polimorfismo possibilita que um mesmo método tenha comportamentos diferentes de acordo com o tipo de objeto utilizado.

Isso facilita a criação de operações que podem ser executadas por diferentes tipos de contas.

### Composição

A composição permite que uma classe seja formada por outros objetos.

Por exemplo, uma conta pode possuir um cliente, um histórico de transações e diferentes produtos financeiros.

## Recursos utilizados

Durante o desafio, foram apresentados recursos importantes do Java:

## Organização do sistema

O sistema pode ser dividido em diferentes responsabilidades:

- **Entidades:** representam clientes, contas, transações e investimentos;
- **Repositórios:** armazenam os dados temporariamente em memória;
- **Serviços:** executam as regras de negócio;
- **Menu:** permite a interação do usuário com a aplicação pelo console.

Essa separação deixa o código mais organizado, facilita a manutenção e melhora o reaproveitamento das funcionalidades.

O desafio apresentou uma visão prática da construção de um sistema bancário utilizando Java e os principais fundamentos da Programação Orientada a Objetos.
